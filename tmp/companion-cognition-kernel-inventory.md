# Companion Cognition Kernel Inventory

This inventory tracks production prompt/policy assembly paths, direct LLM prompt builders, memory-use policy shortcuts, action/response policy shortcuts, commitment/attention policy paths, and their migration decisions for the companion cognition kernel rollout.

## Kickoff State

- Base: `origin/main` after PR #2000, commit recorded by the branch history.
- Draft PR must exist before broad implementation begins.
- This file starts as the inventory artifact and will be expanded as code inspection identifies each path.

## Allowed Boundaries

| Boundary | Why allowed | Guard evidence |
| --- | --- | --- |
| Provider adapters and model request renderers | They convert already assembled typed context/tool definitions into provider payloads. They do not own companion memory-use, commitment, authority, or action policy. | `src/interface/chat/model-request-builder.ts`, `src/orchestrator/execution/agent-loop/agent-loop-model-client.ts`, `src/tools/query/*/prompt.ts`. |
| Grounding renderers | They render typed grounding bundle sections, not companion cognition decisions. | `src/interface/chat/grounding.ts`, `src/grounding/renderers/prompt-renderer.ts`. |
| Diagnostics/operator surfaces | They may expose refs and policy internals only on operator/debug surfaces. Normal surfaces must use redacted projection. | `src/interface/cli/commands/cognition-replay.ts`, `src/runtime/visibility/cognitive-replay-index.ts`, `src/runtime/personal-agent/normal-surface-projection.ts`. |
| Eval fixtures and tests | They may construct prompts/fixtures to exercise behavior but cannot become production caller policy. | `tests/**`, `src/**/__tests__/**`, `tests/slow/**`. |
| Migrations and compatibility helpers | They may recognize legacy storage or exact protocol shapes, but not classify freeform user intent. | `src/runtime/schedule/legacy-cron-migration.ts`, database/public-contract guard scripts. |
